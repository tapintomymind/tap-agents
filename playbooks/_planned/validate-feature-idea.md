# Playbook (STUB) — Validate Feature Idea

Workflow for evaluating whether a proposed feature should be built.

## Build When

You've shipped at least one MVP and want to systematically evaluate whether to build a proposed v2 feature.

## Provisional Purpose

Different from `seed-to-mvp` (which assumes greenfield). This handles the more common steady-state question: "should we build feature X for our existing product?"

## Provisional Steps

1. Capture feature idea (mini-Intake)
2. Critic reviews against existing PRD — does this fit current product, or is it scope drift?
3. Strategist sketches PRD addendum
4. Architect estimates effort + impact on existing tech-strategy
5. Hard checkpoint: build / defer / reject

## Prerequisites

- At least one project at `shipped` or beyond
- Active Strategist + Architect + Critic (founding team)
