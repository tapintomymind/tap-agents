# Playbook (STUB) — Post-Launch Retro

Structured retro after a project has been live and measured for a while.

## Build When

First project transitions to `measured` with at least 2 weeks of data.

## Provisional Purpose

Different from project-end `retro` phase (which captures lessons immediately). This is post-measurement: did the live product match PRD predictions? What surprised us?

## Provisional Steps

1. Collect actual metrics vs PRD predictions
2. Growth Analyst (when activated) provides analysis
3. Feedback Synthesizer (when activated) summarizes user reactions
4. Critic + Org Designer extract lessons
5. Lessons → `memory/lessons-learned.md` with provenance
6. Decisions about next steps (continue / pivot / kill)

## Prerequisites

- Project at `measured` phase
- At least 2 weeks of measurement data
- Ideally: Growth Analyst + Feedback Synthesizer activated

Wait until first project shows measured data. Build playbook against that real situation.
